# cmux-win Phase 2: Core UI — 구현 계획 v1

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.
> **선행 조건:** Phase 1 완료 (147 tests, 9 commits)
> **설계안 정본:** `2026-03-19-cmux-win-design-v3.md` §Phase 2

**Goal:** 사이드바, 분할 패널, 탭, 키보드 단축키가 완전히 동작하는 UI를 구축한다. 사용자가 워크스페이스를 만들고, 터미널을 분할하고, 탭을 전환하고, 단축키로 조작할 수 있어야 한다.

**Architecture:** React 컴포넌트 트리가 SOT(Main Process)의 상태를 구독하고, 사용자 조작을 Action으로 변환하여 SOT에 전달한다. 분할 레이아웃은 PanelLayoutTree(재귀 이진 트리)를 CSS Grid로 렌더링한다. 모든 UI 상태 변경은 SOT를 경유한다.

**Tech Stack:** React 18.3, CSS Grid/Flexbox, react-virtuoso (사이드바 가상화), @dnd-kit (드래그), SOT Action dispatch

## Task 10: SOT — panel.split / panel.resize 구현

**Files:**
- Modify: `src/main/sot/store.ts` (applyAction에 panel.split, panel.resize 구현)
- Test: `tests/unit/sot/panel-operations.test.ts`

### 핵심 로직: panel.split

```
현재 상태:
  workspace.panelLayout = { type: 'leaf', panelId: 'P1' }

panel.split({ panelId: 'P1', direction: 'horizontal', newPanelType: 'terminal' })

결과:
  workspace.panelLayout = {
    type: 'split',
    direction: 'horizontal',
    ratio: 0.5,
    children: [
      { type: 'leaf', panelId: 'P1' },      // 기존 패널
      { type: 'leaf', panelId: 'P2-new' },   // 새 패널
    ]
  }
  + panels에 P2-new 추가
  + surfaces에 새 터미널 서피스 추가
```

### 핵심 로직: panel.resize

```
panel.resize({ panelId: 'P1', ratio: 0.7 })

→ panelLayout 트리에서 P1을 포함하는 split 노드를 찾고
→ 해당 split의 ratio를 0.7로 변경
```

### 테스트

```typescript
describe('panel.split', () => {
  it('splits a leaf into horizontal split with two children');
  it('splits a leaf into vertical split');
  it('splits an already-split layout (nested split)');
  it('creates new panel and surface for the split');
  it('rejects split on nonexistent panel');
});

describe('panel.resize', () => {
  it('resizes a split containing the target panel');
  it('clamps ratio to 0.1-0.9');
  it('rejects resize on leaf-only layout');
});
```

### PanelLayout.tsx — 재귀 CSS Grid 렌더링

```
PanelLayoutTree를 받아서:
  - leaf → PanelContainer 렌더링
  - split → CSS Grid 2칸 (direction에 따라 row/column)
    + PanelDivider (드래그 가능)
    + 재귀적으로 children 렌더링

CSS Grid 전략:
  horizontal split → grid-template-columns: {ratio}fr {1-ratio}fr
  vertical split   → grid-template-rows: {ratio}fr {1-ratio}fr
```

### PanelDivider.tsx — 드래그 리사이즈

```
마우스 드래그로 분할 비율 조정:
  onMouseDown → document.addEventListener('mousemove')
  mousemove → 부모 컨테이너 대비 마우스 위치 → 새 ratio 계산
  onMouseUp → dispatch(panel.resize)
  최소/최대 비율: 0.1 ~ 0.9
```

### PanelContainer.tsx — 패널 타입별 렌더링

```
panelType에 따라:
  'terminal' → XTermWrapper
  'browser'  → placeholder (Phase 3)
  'markdown' → placeholder (Phase 3)

+ PanelTabBar (탭이 2개 이상일 때 표시)
+ 줌 상태면 해당 패널만 full-size 렌더링
```

### shortcuts.ts — 30개 기본 단축키 정의

```typescript
export interface ShortcutDefinition {
  id: string;
  label: string;
  defaultKey: string;    // 'Ctrl+Shift+N' 형식
  action: () => Action;  // SOT Action 생성 함수
}

기본 정의 (설계안 §10 KeyboardShortcutSettings 참조):
  - toggleSidebar: Ctrl+B
  - newWorkspace: Ctrl+N
  - newWindow: Ctrl+Shift+N
  - closeWorkspace: Ctrl+Shift+W
  - closeWindow: Ctrl+Ctrl+W (또는 Ctrl+Alt+W)
  - nextWorkspace: Ctrl+Tab
  - prevWorkspace: Ctrl+Shift+Tab
  - splitRight: Ctrl+D
  - splitDown: Ctrl+Shift+D
  - focusLeft/Right/Up/Down: Ctrl+Alt+Arrow
  - toggleZoom: Ctrl+Shift+Enter
  - newSurface: Ctrl+T
  - closeSurface: Ctrl+W
  - commandPalette: Ctrl+Shift+P (Phase 6)
  ...
```

### useShortcuts.ts — 키 이벤트 바인딩

```
useEffect에서:
  window.addEventListener('keydown', handler)
  handler: 키 조합을 파싱 → shortcuts 매핑에서 찾기 → dispatch(action)
  커스텀 단축키: settings.shortcuts에서 오버라이드 가능
  충돌 감지: 같은 키 조합에 2개 액션 → 경고
```
