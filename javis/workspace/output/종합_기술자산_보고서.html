공통점

A. 단계적 파이프라인 아키텍처

3개 프로젝트 모두 다단계 순차 파이프라인으로 설계. 설교에디터 9단계, 글로벌뉴스 8단계(Stage 1~8), 환경스캐닝 7단계(5-Layer).

B. JSON/YAML 중심 설정 구조

설교에디터 slots.json, 글로벌뉴스 state.yaml, 환경스캐닝 sources*.yaml 등 설정 파일이 Single Source of Truth 역할.

C. 품질 검증 게이트

설교에디터 ~35개, 글로벌뉴스 ~50+개, 환경스캐닝 145+개 자동 검증 게이트. 종료 코드 기반 품질 관리 체계.

D. 대시보드 시각화 계층

설교에디터 Streamlit, 글로벌뉴스 Streamlit 6탭, 환경스캐닝 D3.js 인터랙티브 시그널 맵.

E. 다국어/다문자 처리

한글+헬라어+히브리어(설교에디터), 14개 언어(글로벌뉴스), EN/KO 이중언어+11개 소스(환경스캐닝).

차이점

05 향후 활용 제안

단기 (즉시 ~ 1개월)
제안 1
공용 유틸리티 라이브러리 추출
3개 프로젝트 공통 코드를 javis-common 패키지로 추출. doc_gen / pptx_gen / preview / validation / text_render 모듈 포함. 버그 수정이 모든 프로젝트에 전파.
제안 2
Streamlit 선택기 범용화
image_selector.py를 "다중 후보 선택기" 프레임워크로 범용화. 이미지(설교), 기사 품질(글로벌뉴스), 시그널 큐레이션(환경스캐닝) 모드 지원.
제안 3
검증 게이트 표준화
3개 프로젝트의 검증 패턴을 QualityGate 통합 프레임워크로 표준화. validate() → 0=통과, 1=경고, 2=차단.
중기 (1~3개월)
제안 4
글로벌뉴스 NLP → 환경스캐닝 이식 (S5)
Model2Vec 500x 가속으로 LLM API 비용 절감. SBERT 다국어 임베딩으로 Cross-WF 유사도 비교 고도화. SQLite FTS5+vec로 시그널 DB 즉시 검색.
제안 5
PPT/DOCX 보고서 자동 생성 (S2)
글로벌뉴스 분석 결과 → 슬롯 JSON → sermon_pptx.py로 경영진용 PPT. 환경스캐닝 보고서 → md_to_docx_v2.py로 인쇄용 DOCX.
제안 6
AgenticWorkflow DNA 확산
R→P→I 3단계 프로세스 → 설교에디터 전자동 강화. Context Preservation → 세션 중단/재개 복원력. Safety Hooks → 파괴적 명령 사전 차단.
장기 (3개월+)
제안 7
통합 대시보드 플랫폼
설교 워크플로우 + 글로벌 뉴스 트렌드 + 환경스캐닝 시그널 맵 + 크로스프로젝트 인사이트를 단일 Streamlit 대시보드로 통합.
제안 8
Javis 스킬 생태계
재사용 패턴을 Claude Code 스킬로 패키징: pptx-gen, docx-gen, news-crawl, signal-detect, env-scan, quality-gate.
