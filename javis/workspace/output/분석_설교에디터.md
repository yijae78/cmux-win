## 1. 워크플로우 파이프라인 구조

### 1.1 전체 파이프라인 개요

```
[입력] 성경 본문 + 날짜 + 모드(전자동/단계별)
  │
  ▼
[1] 폴더 생성 ─── sermons/YYYYMMDD-시리즈회차-제목/
  │
  ▼
[2] 기본틀 작성 ─── 기본틀.md (8~25줄 뼈대, Q1-Q2 위계)
  │                  스킬: sermon-writer (서신=강해형, 내러티브=N패키지)
  │                  템플릿: templates/기본틀-template.md (서신) / 기본틀-template-내러티브.md
  ▼
[3] 전체 원고 작성 ── 원고.md (8,000~12,000자)
  │                   스킬: sermon-writer (v3.1 표준, 10대 핵심 원칙)
  │                   검증: validate_manuscript.py (28항목)
  ▼
[4] 슬롯 메타 추출 + 동기화
  │  ├─ extract_pp_context.py → slots_raw.json (▶PP 마커 파싱)
  │  ├─ Claude가 slots.json 보완 (desc, context, text, has_image)
  │  ├─ sync_pp_titles.py (슬롯 ↔ 원고 PP제목 양방향 동기화)
  │  ├─ generate_text_slides.py → images/text_PPNN.png (본문 슬롯)
  │  └─ validate_slots.py (무결성 검증, 종료코드 2=차단)
  ▼
[5] 이미지 생성 ── NLM 6벌 병렬 (nlm-image-maker 스킬)
  │  ├─ 비주얼 브리프 작성 (영어 시각 묘사)
  │  ├─ NLM 노트북 6개 병렬 생성 + focus 프롬프트
  │  ├─ PPTX 다운로드 → extract_pptx_images.py → slide_NN.png
  │  └─ 내용 기반 슬롯 매핑 (순차 매핑 절대 금지)
  ▼
[6] 이미지 선택 ── image_selector.py (Streamlit :8503)
  │  ├─ 2-Agent 교차 검수 (7점 미만 = null)
  │  ├─ 브라우저 선택/재생성 루프
  │  └─ [🎉 최종 컨펌] → _selection.json (status=confirmed)
  ▼
[7] 컨펌 후 자동처리 ── post_confirm.py
  │  ├─ ① 원고에 이미지 마크다운 삽입
  │  ├─ ② sync_pp_titles 동기화
  │  ├─ ③ md_preview.py → preview.html (base64 인라인)
  │  └─ ④ 브라우저 자동 오픈 → 신교수님 검수
  ▼
[8] DOCX 변환 ── md_to_docx_v2.py --clean-v2
  │              스타일: 나눔명조 15pt, 장평 85%, 자간 -3%
  │              박스 5종: scripture/benediction/pp/monologue/proclamation
  ▼
[9] PPT 생성 ── sermon_pptx.py (전 부서 공용)
  │             표지 3장 + 슬롯별 이미지/네이티브 텍스트 분기
  │             글꼴 임베딩 포함
  ▼
[완료] output/ 폴더에 .docx + .pptx 저장
```

## 3. 이미지 생성 전략

### 3.1 이미지 엔진 3계층

```
┌─────────────────────────────────────────────────────────┐
│                    이미지 생성 전략                       │
├─────────────────┬───────────────────┬───────────────────┤
│ NLM (노트북LM)   │ Pollinations.ai   │ generate_text_    │
│ ⭐ 메인 엔진     │ 보조/대체         │ slides.py         │
│                 │                   │ 텍스트 전용        │
├─────────────────┼───────────────────┼───────────────────┤
│ 장면/원어/감성   │ 장면 이미지 (무료) │ 성경 본문/대비표   │
│ 슬라이드 이미지  │ API 키 불필요     │ 키워드/축도        │
│ 6벌 병렬        │ 16:9 1920x1080   │ 1280x720 PNG     │
│ 10회/일 무료    │ 15초/요청 제한    │ 무제한            │
└─────────────────┴───────────────────┴───────────────────┘
```

### 3.2 NLM 이미지 생성 상세 (메인 엔진)

**확정 방침 (2026-05-31)**: 이미지는 무조건 NLM. 유료 모델(ChatGPT/Gemini)은 신교수님 명시 요청 시에만.

**선택 이유**: "잘 그린 그림 ≠ 설교에 맞는 그림" — NLM의 절제·여백·신비·톤 일관성이 설교 슬라이드에 더 적합.

#### 3가지 스타일 DNA (20260325 WITH 설교 추출)

| 스타일 | 비중 | 특징 | 용도 |
|--------|------|------|------|
| **A. 초실사 시네마틱** | 70% | 8K DSLR, 다크 배경, 극적 역광, 황금빛 | 장면 슬라이드 |
| **B. 원어 상징 판타지** | 25% | 원어가 장면 안 발광 오브젝트 (돌 각인, 불꽃) | 원어 분석 슬라이드 |
| **D. 감성적 인물** | 5% | 한국인, 황금 역광, 포옹/눈물 | 감성 장면 |

#### 6벌 병렬 생성 프로세스

```
[1] 비주얼 브리프 작성 (원고 → 영어 시각 묘사 정제)
     │
[2] NLM 노트북 6개 생성 (A~F, 각각 별도 노트북)
     │  A=A-WARM  B=B-GLOW  C=C-DRAMA  D=A-EPIC  E=B-FIRE  F=C-TENDER
     │
[3] 각 노트북에 비주얼 브리프 소스 추가 + focus 프롬프트 전송
     │
[4] 병렬 slides create (7~10분, 순차 대비 50% 단축)
     │
[5] PPTX 다운로드 → extract_pptx_images.py → slide_NN.png
     │
[6] 내용 기반 슬롯 매핑 (Read tool로 시각 확인, 순차 매핑 금지)
     │
[7] image_selector.py (Streamlit :8503)에서 선택/재생성
```

#### 인물 민족 슬롯별 분기 규칙

| 장면 유형 | 인물 설정 |
|----------|----------|
| 성경 시대 (예수·베드로·모세 등) | 1세기 유대인/셈족 외모 |
| 현대 메타포/예화 | 한국인 디폴트 |
| 추상/상징 (빛, 바람) | 인물 없음/실루엣 |

### 3.3 generate_text_slides.py (텍스트 슬라이드)

성경 본문, 대비표, 키워드, 축도 등 **텍스트 중심 슬라이드**를 NLM 대신 자동 렌더링.

| 항목 | 사양 |
|------|------|
| 캔버스 | 1280×720px |
| 배경 | 검정 + 좌측 황금 바 + 진네이비 헤더 밴드 |
| 폰트 시작 | 80pt → floor 56pt (4pt 단위 축소) |
| 히브리어 | HarfBuzz 셰이핑 (uharfbuzz + freetype-py) |
| 줄간격 | 4줄 이하 +18 / 5~6줄 +14 / 7줄+ +10 |
| 출력 | images/text_PPNN.png |

## 4. 재사용 가능 패턴 식별

### 4.1 슬롯 기반 콘텐츠 파이프라인 패턴

**패턴**: 원고(MD) → 슬롯 메타(JSON) → 이미지/텍스트 분기 → 선택 → 문서 변환

```
[원고.md]
    │ extract_pp_context.py
    ▼
[slots.json] ← 단일 진실 소스 (Single Source of Truth)
    │
    ├── has_image: true  → NLM/외부 이미지
    ├── has_image: false → generate_text_slides.py
    │
    ▼
[image_selector] → [post_confirm] → [DOCX/PPT]
```

**적용 가능 프로젝트**: 교육 콘텐츠 생성, 프레젠테이션 자동화, 보고서 파이프라인

### 4.2 다중 스타일 병렬 생성 + 선택 패턴

**패턴**: 하나의 브리프에서 N가지 스타일로 병렬 생성 → 슬롯별 최적 선택

```
[비주얼 브리프 1개]
    │
    ├── 스타일 A (노트북 A)  ──┐
    ├── 스타일 B (노트북 B)  ──┤
    ├── 스타일 C (노트북 C)  ──┼── 병렬 생성
    ├── 스타일 D (노트북 D)  ──┤
    ├── 스타일 E (노트북 E)  ──┤
    └── 스타일 F (노트북 F)  ──┘
                                  │
                            [선택기 대시보드]
                                  │
                            [최적 조합 선정]
```

**적용 가능**: A/B 테스트, 디자인 변형 비교, 마케팅 크리에이티브 선택

### 4.3 양방향 동기화 패턴

**패턴**: 두 데이터 소스(원고 ↔ 슬롯 JSON) 간 비파괴적 양방향 동기화

```
sync_pp_titles.py --from slots --to manuscript   (슬롯 → 원고)
sync_pp_titles.py --from manuscript --to slots    (원고 → 슬롯)
```

- 백업 자동 생성 (.bak)
- --dry-run 지원
- 변경 발생 시점마다 자동 실행

**적용 가능**: 설정/문서 동기화, CMS↔코드 동기화, 스키마↔문서 정합

### 4.4 3단계 검증 게이트 패턴

**패턴**: 파이프라인 진행 전 자동 검증, 종료 코드로 게이트 제어

```
[1] validate_manuscript.py (28항목, 원고 품질)
    → 종료코드 0=통과 / 1=경고 / 2=차단

[2] validate_slots.py (4항목, 슬롯↔이미지 정합)
    → 종료코드 0=무결 / 2=NLM 토큰 소비 금지

[3] 2-Agent 교차 검수 (이미지 선택기)
    → 7점 미만 = null 처리 (강제 매핑 금지)
```

**적용 가능**: CI/CD 게이트, 데이터 파이프라인 품질 체크, 컨텐츠 퍼블리싱

### 4.5 디자인 토큰 중앙 관리 패턴

**패턴**: 모든 시각 설정을 단일 파일(style.py/design.py)에 집중 관리

```python
# docx_style/style.py — DOCX용 디자인 토큰
STYLE = { COLOR_H1, SIZE_BODY, FONT_BODY, PAGE_WIDTH_CM, ... }

# kids_pptx_design.py — 아동부 PPT 디자인 토큰
FONT_BODY, FONT_HEADER, SLIDE_WIDTH, SIZE_BODY, WHITE, YELLOW, ...
```

**적용 가능**: 브랜드 일관성 관리, 멀티 포맷 출력 (DOCX/PPT/HTML), 테마 전환

### 4.6 Streamlit 선택기 + _trigger.json 비동기 패턴

**패턴**: 브라우저 UI에서 액션 → JSON 트리거 파일 생성 → 백엔드 감지 → 처리 → 파일 삭제

```
[브라우저] 재생성 버튼 클릭
    │
    ▼
[_trigger.json] 생성 (슬롯 + 프롬프트)
    │
    ▼
[auto_regen.py / Claude] 감지 → NLM 재생성
    │
    ▼
[_trigger.json] 삭제 → 선택기 자동 새로고침
```

**적용 가능**: Streamlit 기반 승인/워크플로우 도구, 파일 기반 이벤트 시스템

### 4.7 부서 분기 + 공용 엔진 패턴

**패턴**: 부서별로 스킬/디자인 상수만 분리, 핵심 엔진은 공유

```
sermon_pptx.py (공용 PPT 엔진)
    ├── 장년부: pptx_text_native.py (디자인)
    ├── 아동부: kids_pptx_builder.py + kids_pptx_design.py (독립 디자인)
    └── 새벽:   pptx_text_native.py (장년부와 공유)

generate_text_slides.py (공용 텍스트 렌더러)
    └── 모든 부서가 동일 스크립트 사용 (밴드 색상만 slots.json으로 override)
```

**적용 가능**: 멀티 브랜드/테넌트 콘텐츠 시스템, 화이트라벨 앱

### 5.1 의존성

| **lxml** | PPT 애니메이션 XML 주입 |
| **nlm CLI** | 노트북LM 슬라이드 생성 |

### 5.3 아키텍처 특징

1. **슬롯 JSON 중심 설계**: slots.json이 파이프라인 전체의 단일 진실 소스
2. **스킬 기반 오케스트레이션**: Claude Code 스킬이 워크플로우 라우팅 담당
3. **파일 기반 이벤트**: _trigger.json, _selection.json으로 비동기 통신
4. **버전 보존 문화**: v2/v3 백업, _archive, _backup 파일 다수 보존
5. **전 부서 공용화 진행**: sermon_pptx.py로 PPT 엔진 통합 (kids-ppt-maker 폐기)
